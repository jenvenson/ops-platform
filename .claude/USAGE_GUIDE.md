# Claude Skills and Subagents 使用指南

## 一、Skills（技能）使用说明

### 什么是Skills？
Skills是Claude AI助手的特定功能模块，每个Skill专注于OPS平台的某个具体功能领域。当您需要Claude执行特定任务时，可以明确指定使用某个Skill。

### 如何使用Skills？

#### 1. 直接调用语法
```
"使用 [skill名称] 做 [具体任务]"
```

**示例：**
- `"使用 ops-cmdb 技能添加一个新服务器到开发环境"`
- `"使用 ops-deployment 技能部署我的应用到预发布环境"`
- `"使用 ops-security 技能扫描 192.168.1.0/24 网段的安全漏洞"`

#### 2. 技能列表及用途

| 技能名称 | 用途 | 适用场景 |
|---------|------|----------|
| `ops-cmdb` | CMDB管理 | 管理项目、环境、服务器等基础设施配置 |
| `ops-deployment` | 部署管理 | 应用部署、发布、归档等操作 |
| `ops-jenkins` | Jenkins管理 | CI/CD流水线、作业管理 |
| `ops-consul` | Consul配置管理 | 服务发现、配置中心管理 |
| `ops-security` | 安全扫描 | 漏洞扫描、资产管理、工单处理 |
| `ops-monitoring` | 系统监控 | 性能指标查看、主机监控 |
| `ops-alerting` | 告警管理 | 告警规则、通知渠道、联系人管理 |
| `ops-admin` | 系统管理 | 用户、角色、权限管理 |

#### 3. Skills使用步骤
1. **明确需求**：确定您想要执行的具体操作
2. **选择合适技能**：根据操作类型选择对应的Skill
3. **提供必要参数**：在指令中包含操作所需的具体信息
4. **等待执行结果**：Claude将执行操作并返回结果

#### 4. 使用示例
- **CMDB管理**：
  ```
  "使用 ops-cmdb 技能创建一个名为 '电商项目' 的新项目，描述是 '电商平台后端服务'"
  ```

- **部署操作**：
  ```
  "使用 ops-deployment 技能部署 'api-service' 应用到 'production' 环境"
  ```

- **安全扫描**：
  ```
  "使用 ops-security 技能开始对 '10.0.0.0/24' 网络段的安全扫描"
  ```

## 二、Subagents（子代理）使用说明

### 什么是Subagents？
Subagents是自主运行的智能代理，它们在后台持续工作，主动监控和执行特定任务，无需手动调用。

### Subagents列表及功能

#### 1. 代码审核代理 (`code-review-agent`)

**功能**：专家级代码审查专员，主动审查代码质量、安全性和可维护性

**何时激活**：
- 在代码编写或修改后立即自动启动
- 创建Pull Request时自动运行
- 代码提交后自动检查

**检查内容**：
- 代码质量评估（结构、可读性、性能）
- 安全漏洞检测（SQL注入、XSS、CSRF等）
- 最佳实践验证
- 代码复杂度分析
- 文档和注释检查

#### 2. 测试验证代理 (`test-validation-agent`)

**功能**：专业QA和测试工程师，通过自动化测试策略确保软件可靠性

**何时激活**：
- 代码变更时自动运行
- 新功能开发过程中
- 修复Bug后自动验证
- 部署前自动执行回归测试

**验证内容**：
- 设计全面的测试套件
- 识别边界条件和异常场景
- 验证代码正确性
- 生成单元、集成和端到端测试
- 分析测试覆盖率

#### 3. 审计合规代理 (`audit-compliance-agent`)

**功能**：跟踪系统操作，确保合规性

**何时激活**：
- 持续监控所有系统操作
- 定期生成合规报告
- 发现违规行为时自动告警

**监控内容**：
- 跟踪和记录所有系统操作
- 验证安全政策合规性
- 识别政策违规行为
- 维护详细的审计轨迹

### 如何与Subagents互动？

#### 1. 查看Subagents状态
```
"检查代码审核代理的状态"
"查看测试验证代理最近的分析结果"
```

#### 2. 请求特定报告
```
"请代码审核代理生成最新的代码质量报告"
"让测试验证代理提供测试覆盖率分析"
```

#### 3. 配置Subagents行为
```
"设置代码审核代理对安全问题更敏感"
"配置测试验证代理增加性能测试覆盖"
```

## 三、最佳实践

### Skills使用最佳实践：
1. **明确指定**：使用清晰具体的指令指定所需的Skill
2. **提供参数**：包含所有必需的参数信息
3. **分步操作**：复杂的操作可以分步骤执行
4. **确认结果**：检查操作结果是否符合预期

### Subagents最佳实践：
1. **设置阈值**：为各种检测设置合适的告警阈值
2. **定期审查**：定期查看Subagents的报告和建议
3. **及时响应**：对Subagents发现的问题及时处理
4. **优化配置**：根据实际需求调整Subagents的行为参数

## 四、常见问题

**Q: 我可以同时使用多个Skills吗？**
A: 可以，您可以通过顺序调用或请求Claude协调多个Skills完成复杂任务。

**Q: Subagents会消耗很多系统资源吗？**
A: Subagents经过优化设计，只在需要时执行关键检查，资源消耗相对较低。

**Q: 如何知道某个Subagent正在运行？**
A: 您可以通过询问Claude或查看系统日志来了解Subagents的运行状态。

**Q: 我可以自定义Subagents的检查规则吗？**
A: 可以，通过配置文件可以调整Subagents的行为和检查规则。
