# 安装包聚合打包功能需求文档

## 1. 功能概述
安装包聚合打包功能是针对运维管理平台的一个新增功能模块，旨在提供集中化的应用安装包聚合打包服务。该功能允许用户通过简单的交互界面，选择多个应用项目，自动触发Jenkins的聚合打包构建任务，并提供打包状态跟踪和结果归档功能。

## 2. 业务需求

### 2.1 功能需求
- **项目选择**：用户可以从系统中已有的项目列表中选择一个项目
- **应用选择**：在选定项目下，用户可以选择多个应用进行聚合打包
- **应用筛选**：支持按应用名称进行模糊搜索和筛选
- **批量选择**：支持全选、取消全选、反选等批量操作
- **打包参数配置**：支持选择打包类型（全部、前端、后端）
- **任务触发**：触发Jenkins进行聚合打包构建
- **状态跟踪**：实时跟踪打包任务执行状态
- **结果展示**：展示每个应用的打包结果和状态

### 2.2 非功能需求
- **安全性**：遵循系统现有的权限控制机制
- **稳定性**：支持并发打包任务，避免单点故障
- **可扩展性**：预留接口以支持更多打包策略和参数配置
- **易用性**：提供直观的UI界面，减少用户操作复杂度

## 3. 用户角色
- **运维工程师**：负责发起和监控聚合打包任务
- **项目经理**：查看打包结果和状态

## 4. 功能流程

### 4.1 主流程
1. 用户进入"自动化部署" → "安装包聚合打包"页面
2. 选择要进行聚合打包的项目
3. 从项目下的应用列表中选择需要打包的应用
4. 配置打包参数（如打包类型：全部、前端、后端）
5. 点击"开始聚合打包"按钮
6. 系统触发Jenkins进行聚合打包构建
7. 页面展示打包进度和状态
8. 打包完成后展示结果

### 4.2 状态跟踪流程
1. 用户发起打包任务后，系统记录任务状态为"排队中"
2. 当Jenkins接收任务后，状态变为"构建中"
3. 系统定时轮询Jenkins获取构建状态
4. 构建完成后更新状态为"成功"或"失败"
5. 如果失败，显示失败原因

## 5. 输入输出

### 5.1 输入参数
- **项目名称**：选择要进行聚合打包的项目
- **应用列表**：选择参与聚合打包的应用集合
- **打包类型**：all（全部）、frontend（前端）、backend（后端）
- **打包描述**：可选的打包任务描述信息

### 5.2 输出结果
- **打包任务状态**：排队中、构建中、成功、失败
- **各应用打包结果**：显示每个应用的打包状态
- **错误信息**：构建失败时显示详细错误信息
- **打包产物信息**：构建成功后显示产物下载链接

## 6. 接口需求

### 6.1 前端页面
- 项目选择下拉框
- 应用选择表格（支持勾选）
- 打包参数配置表单
- 打包状态展示页面
- 结果展示表格

### 6.2 后端API
- 获取项目列表接口
- 获取项目下应用列表接口
- 触发聚合打包接口
- 查询打包任务状态接口
- 获取打包结果接口
- 获取打包历史记录接口

## 7. 依赖项
- Jenkins服务：提供构建能力
- 数据库：存储打包任务记录和结果
- 现有用户权限系统：确保功能访问安全

## 8. 异常处理
- 项目不存在：提示用户重新选择
- 应用配置缺失：提示具体哪个应用缺少必要配置
- Jenkins连接失败：提示服务不可用
- 打包失败：记录失败原因并展示给用户

## 9. 性能要求
- 页面响应时间：小于2秒
- 任务状态轮询间隔：3-5秒
- 支持并发打包任务数：至少5个

## 10. 安全考虑
- 遵循现有权限控制体系
- 对用户输入进行校验和清理
- 敏感操作需要二次确认
