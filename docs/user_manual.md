# 运维管理平台用户使用手册

## 1. 系统概述

运维管理平台是一个集中化的管理平台，提供资产中心、变更发布、监控中心、告警事件、安全中心、系统管理等功能。用户可以通过统一的界面管理多个项目和环境。

### 1.1 平台架构
- **资产中心**：项目、环境、主机、应用流水线管理 (路径: /cmdb/projects, /cmdb/environments, /cmdb/servers, /cmdb/applications)
- **变更发布**：迭代部署、部署记录、归档打包、聚合打包、Consul配置变更、Jenkins任务 (路径: /deploy/release, /deploy/history, /deploy/archive, /deploy/aggregate-package, /consul/config, /jenkins/views)
- **监控中心**：监控大屏、概览、Grafana集成 (路径: /monitor/bigscreen, /monitor/overview, /monitor/dashboards)
- **平台事件中心**：跨模块统一查看部署、归档、告警、助手等事件 (路径: /platform/events)
- **平台审计**：操作日志、访问日志、登录日志审计 (路径: /platform/audit)
- **告警事件**：事件中心、规则、联系人管理 (路径: /alarm/events, /alarm/rules, /alarm/contacts)
- **安全中心**：安全扫描、安全资产、漏洞管理、文件完整性巡检 (路径: /security/tasks, /security/assets, /security/vulnerabilities, /security/fim/*)
- **系统管理**：用户、角色、菜单管理 (路径: /admin/users, /admin/roles, /admin/menus)

### 1.2 登录与认证
首次访问系统时，请使用您的账户凭据登录。系统采用多因素认证，确保安全性。

## 2. 资产中心

### 2.1 项目管理 (/cmdb/projects)
项目是所有其他资源的顶层容器，所有环境、服务器都属于某个项目。

#### 创建项目
1. 导航至 `/cmdb/projects`
2. 点击"新增项目"按钮
3. 输入项目名称、描述、负责人等信息
4. 设置预算和资源配额
5. 点击"保存"

#### 编辑项目
1. 在项目列表中选择目标项目
2. 点击"编辑"按钮
3. 修改所需信息
4. 点击"保存"

#### 删除项目
注意：删除项目将同时删除该项目下的所有环境和资源，请谨慎操作。

### 2.2 环境管理 (/cmdb/environments)
环境管理模块允许创建不同的部署环境，如开发、测试、预生产、生产环境。

#### 创建环境
1. 导航至 `/cmdb/environments`
2. 点击"新增环境"按钮
3. 选择所属项目
4. 输入环境名称（如dev、test、prod）
5. 配置环境参数和访问权限
6. 点击"保存"

#### 环境类型
- **开发环境**：用于开发人员日常开发和测试
- **测试环境**：用于QA团队的功能验证
- **预生产环境**：模拟生产环境进行最终验证
- **生产环境**：线上正式运行环境

### 2.3 服务器管理 (/cmdb/servers)
服务器管理模块用于管理所有物理机、虚拟机、容器等基础设施资源。

#### 添加服务器
1. 导航至 `/cmdb/servers`
2. 点击"添加服务器"按钮
3. 选择服务器类型（物理机、虚拟机、容器）
4. 输入服务器基本信息（主机名、IP地址、操作系统等）
5. 配置连接参数和访问凭证
6. 标记服务器角色和用途
7. 点击"保存"

#### 服务器属性
- 主机名
- IP地址
- 操作系统类型及版本
- CPU和内存配置
- 磁盘空间
- 所属环境和项目

### 2.4 应用流水线管理 (/cmdb/applications)
应用流水线管理页面用于维护应用与 Jenkins 流水线的关联信息，便于后续迭代部署、归档和批量导入。

#### 新增应用流水线
1. 导航至 `/cmdb/applications`
2. 点击"新增应用流水线"按钮
3. 输入应用名称
4. 选择所属项目
5. 选择项目环境
6. 按需填写 Jenkins 发布流水线、Jenkins 归档流水线和描述
7. 点击"保存"

#### 从 Jenkins 导入流水线
1. 导航至 `/cmdb/applications`
2. 点击"从 Jenkins 导入流水线"
3. 输入视图名称并获取 Jobs 预览
4. 如 Job 名带有环境号、版本号或自定义前缀，可在"应用名前缀清理规则"中输入要去掉的前缀，例如 `190-`、`V2.5.1-`
5. 预览中的应用名称会实时按相同规则清洗，导入时也会按相同规则落库

#### 导入命名说明
- 支持从 Job 名中自动去掉常见视图前缀，例如 `prod166-technical-research` -> `technical-research`
- 支持去掉版本型前缀，例如 `fat-70-V2.5.1-firmware-research` 配合 `V2.5.1-` -> `firmware-research`
- 支持同环境前缀加数字号段的场景，例如 `mscs_dev-180-fscr-awd` -> `fscr-awd`

## 3. 变更发布

### 3.1 迭代部署 (/deploy/release)
迭代部署功能用于按项目和应用触发发布，并支持选择部署类型。

#### 部署准备
1. 确保目标环境可用
2. 检查部署资源充足
3. 准备部署包和配置文件

#### 执行部署
1. 导航至 `/deploy/release`
2. 选择需要部署的项目和应用
3. 选择要发布的应用
4. 选择目标环境
5. 选择部署类型（全部部署：前端+后端、前端部署、后端部署）
6. 确认发布信息
7. 点击"开始部署"

#### 部署类型
- **全部部署（前端+后端）**：同时触发前端和后端部署
- **前端部署**：仅部署前端资源
- **后端部署**：仅部署后端服务

### 3.2 归档打包 (/deploy/archive)
归档打包功能用于触发项目下应用的归档任务，并保存对应版本产物，便于后续追溯与回滚。

#### 归档应用
1. 导航至 `/deploy/archive`
2. 选择要归档的项目
3. 选择需要归档的应用，最多支持 3 个应用并发归档
4. 进入确认页后选择归档类型：全部归档、前端归档或后端归档
5. 点击“开始归档”并等待结果
6. 如需查看历史记录，可进入 `/deploy/archived`

#### 归档策略
- 自动归档最近10个版本
- 手动归档重要里程碑版本
- 设置过期策略清理旧版本

### 3.3 归档历史 (/deploy/archived)
归档历史页面用于查看历史归档记录、归档结果和相关版本信息。

#### 查看归档历史
1. 导航至 `/deploy/archived`
2. 查看归档历史记录列表
3. 根据应用名称或归档结果筛选目标记录
4. 按需查看归档详情和版本信息

#### 下载归档产物
1. 导航至 `/deploy/archived`
2. 在归档历史列表中找到目标记录的“下载地址”
3. 点击“查看文件”打开归档文件列表
4. 在文件列表中点击“下载”获取相关归档包，或点击“复制链接”后在新窗口打开下载

### 3.4 部署记录 (/deploy/history)
部署记录页面用于查看最近的部署结果、状态和触发时间。

#### 查看部署记录
1. 导航至 `/deploy/history`
2. 查看部署记录列表
3. 根据应用名称、部署状态和时间定位目标记录
4. 按需刷新列表获取最新部署结果

### 3.5 聚合打包功能 (/deploy/aggregate-package)
聚合打包功能用于一次性打包多个应用，从Consul获取配置并触发Jenkins任务。

#### 执行聚合打包
1. 导航至 `/deploy/aggregate-package`
2. 输入项目名称
3. 在应用名称列表中输入需要打包的应用名称（每行一个）
4. 点击"开始聚合打包"
5. 系统将自动从Consul获取各应用的tag配置，并触发Jenkins任务

#### 打包参数
- **项目名称**：要打包的项目名
- **应用名称列表**：要包含在聚合打包中的应用名称，每行一个
- **Jenkins任务**：系统将使用 `http://your-jenkins-server/view/auto-archive-deploy/job/aggregation/` 任务
- **Consul路径**：系统从 `plugin/aggregation/` 路径获取各应用的tag值

### 3.6 聚合历史 (/deploy/aggregated-history)
聚合历史页面用于查看聚合打包任务的执行结果、状态和下载地址。

#### 查看聚合历史
1. 导航至 `/deploy/aggregated-history`
2. 查看聚合历史记录列表
3. 根据项目名称、Tag、状态或时间筛选目标记录
4. 按需查看日志、进度和下载地址

#### 下载聚合包
1. 导航至 `/deploy/aggregated-history`
2. 在聚合历史列表中找到目标记录的“下载地址”
3. 可直接点击下载链接下载聚合包
4. 如需查看当前聚合产物列表，可点击“查看文件”打开文件列表后再下载或复制链接

### 3.7 Consul配置 (/consul/config)
Consul配置页面用于维护 Consul 实例连接信息，支持新增、编辑、删除、测试连接和设置默认配置。

#### 添加配置
1. 导航至 `/consul/config`
2. 点击"添加配置"按钮
3. 输入配置名称、Consul地址和数据中心
4. 按需填写 Token、用户名和密码
5. 点击保存

#### 常用操作
- **测试连接**：验证当前配置是否可以正常连接到 Consul
- **设为默认**：将该配置设置为默认 Consul 配置

### 3.8 批量配置下发 (/consul/batch-all)
批量配置下发用于按后缀批量复制或替换 Consul Key 内容。

#### 执行批量配置下发
1. 导航至 `/consul/batch-all`
2. 选择 Consul 配置
3. 输入源后缀和目标后缀
4. 按需开启递归复制或原后缀内替换
5. 可选指定"项目选择"，只对选中的项目下发；留空表示处理当前 Consul 配置下的全部项目
6. 根据需要填写 Tag、Server、Branch 等替换规则
7. 点击"执行批量配置下发"
8. 在结果区域查看成功和失败统计

#### 批量下发说明
- 递归复制默认开启，既支持复制目录下的子键，也支持复制形如 `plugin/<项目>/<后缀>` 的单个 Key
- 如果结果显示"成功 0，失败 0"，通常表示当前所选项目或源后缀下没有匹配到可复制的 Key
- 原后缀内替换不会创建新 Key，而是直接修改原后缀下已有内容

## 4. Jenkins管理 (/jenkins/views)

### 4.1 视图复制功能
Jenkins视图复制功能可以将一个视图中的所有Job复制到另一个视图，支持Job名称和Tag值的自动替换。

#### 执行视图复制
1. 导航至 `/jenkins/views`
2. 在"Jenkins地址"字段输入Jenkins服务器地址（默认为 `http://your-jenkins-server`）
3. 输入"源视图名称"（要复制的源视图名称）
4. 输入"目标视图名称"（要创建的目标视图名称）
5. （可选）在"Job名称替换"部分输入原Job名称模式和新Job名称模式
6. （可选）在"流水线Tag值替换"部分输入原Tag值模式和新Tag值模式
7. 点击"执行视图复制"按钮

#### 复制规则说明
- Job 名称替换按字面量处理；如果同一个原模式填写了多条规则，系统以最后一条为准
- 流水线 Tag 值替换只作用于 `tag` 参数上下文，不会误改其他 `defaultValue`
- 例如原 Tag 为 `15s_dev`、新 Tag 为 `prod166` 时，会保持为 `prod166`，不会被截断成 `prod`
- 系统会尝试自动审批 Jenkins `pending scripts`；如果 Jenkins 还存在 `pending signatures` 等其他审批项，仍需到 Script Approval 页面手工处理

#### 复制结果查看
执行完成后，系统将显示复制结果：
- 成功复制的Job数量
- 失败的Job数量
- 跳过的Job数量
- 脚本自动审批的数量

### 4.2 视图删除功能
Jenkins视图删除功能允许您删除指定视图及其中的Job。

#### 删除视图步骤
1. 在"视图删除"区域输入要删除的视图名称
2. 点击"查询视图"按钮
3. 系统将列出该视图下的所有Job
4. 您可以进行以下操作：
   - **删除选中**：删除勾选的Job
   - **仅删除视图**：只删除视图本身，保留所有Job
   - **删除视图及全部Job**：删除整个视图及其中的所有Job

### 4.3 凭据管理功能
Jenkins凭据管理功能用于创建SSH密钥凭据。

#### 创建SSH凭据
1. 在"新增Jenkins凭据"区域填写以下信息：
   - 凭据ID（通常是主机IP地址）
   - 用户名（默认为root）
   - 描述（可选）
   - 私钥内容（完整的SSH私钥）
2. 点击"创建凭据"按钮

## 5. 监控中心

### 5.1 监控大屏 (/monitor/bigscreen)
监控大屏展示关键业务指标和系统运行状态的实时视图。

#### 自定义大屏
1. 导航至 `/monitor/bigscreen`
2. 点击"编辑布局"按钮
3. 拖拽添加监控图表组件
4. 配置图表数据源和展示参数
5. 保存布局

### 5.2 概览 (/monitor/overview)
概览页面提供系统的整体运行状况，包括CPU、内存、磁盘、网络等基础指标，以及应用层面的性能数据。

#### 指标类型
- **系统指标**：CPU使用率、内存占用、磁盘IO、网络流量
- **应用指标**：响应时间、吞吐量、错误率、并发连接数
- **业务指标**：交易成功率、关键业务流程耗时

### 5.3 Grafana集成 (/monitor/dashboards)
平台集成了Grafana，用户可以通过嵌入的Grafana界面创建专业的监控图表和仪表板。

#### 创建仪表板
1. 导航至 `/monitor/dashboards`
2. 点击"新建仪表板"按钮
3. 选择数据源（Prometheus、MySQL等）
4. 添加面板和查询
5. 配置样式和告警规则
6. 保存仪表板

## 6. 告警事件

### 6.1 告警事件 (/alarm/events)
告警事件模块汇总所有触发的告警，包括系统告警和业务告警。

#### 查看告警
1. 导航至 `/alarm/events`
2. 查看告警列表和详细信息
3. 确认或关闭告警
4. 记录处理备注

#### 告警级别
- **严重**：系统不可用或关键功能故障
- **警告**：性能下降或潜在风险
- **信息**：一般性通知

### 6.2 告警规则 (/alarm/rules)
告警规则允许用户自定义监控指标和报警条件。

#### 创建告警规则
1. 导航至 `/alarm/rules`
2. 点击"新建规则"按钮
3. 选择监控指标
4. 设置阈值和触发条件
5. 配置通知渠道和联系人
6. 启用规则

### 6.3 联系人管理 (/alarm/contacts)
联系人管理用于配置告警通知的接收人。

#### 添加联系人
1. 导航至 `/alarm/contacts`
2. 点击"新增联系人"按钮
3. 输入姓名、联系方式（邮箱、手机号）
4. 分配告警级别权限
5. 设置值班时间
6. 点击"保存"

## 7. 安全中心

### 7.1 安全扫描 (/security/tasks)
安全扫描页面用于创建和查看漏洞扫描任务。当前新建入口仅保留网站漏洞和主机漏洞两类任务。

#### 创建网站漏洞扫描
1. 导航至 `/security/tasks`
2. 点击"新建漏洞扫描"
3. 选择"网站漏洞"
4. 输入目标 URL（http/https）
5. 按需填写登录态信息
6. 查看"接入预检查"结果
7. 点击"创建并开始扫描"

#### 创建主机漏洞扫描
1. 导航至 `/security/tasks`
2. 点击"新建漏洞扫描"
3. 选择"主机漏洞"
4. 输入一个或多个目标 IP
5. 点击"创建并开始扫描"

#### 网站漏洞扫描说明
- 当前 Web 扫描主路径已收敛为登录后 Web 扫描，匿名 Web 扫描已禁用
- "接入预检查"会给出 `可直接接入`、`需要定制认证流`、`暂不建议自动扫描` 三类结论
- 如对站点接入方式拿不准，可点击"不确定？问小助手"
- 为控制扫描时长，部分低价值目标只做规则检测，不一定运行完整模板验证

### 7.2 安全资产 (/security/assets)
安全资产集中管理所有的IT资产，包括硬件、软件、许可证等。

#### 资产登记
1. 导航至 `/security/assets`
2. 点击"新增资产"按钮
3. 选择资产类型（硬件、软件、网络设备）
4. 输入资产信息（型号、序列号、采购日期等）
5. 分配责任人和存放位置
6. 点击"保存"

### 7.3 漏洞管理 (/security/vulnerabilities)
漏洞管理模块跟踪和管理发现的安全漏洞，从发现、评估、修复到验证的全生命周期管理。

#### 漏洞处理流程
1. 发现漏洞（自动扫描或人工上报）
2. 评估风险等级和影响范围
3. 分配修复任务和截止日期
4. 实施修复措施
5. 验证修复效果
6. 关闭漏洞记录

### 7.4 文件完整性巡检 (/security/fim/*)
文件完整性巡检（FIM）用于监控服务器上的关键文件变化，确保系统安全。

#### FIM 策略管理 (/security/fim/policies)
管理 FIM 监控策略，包括配置监控路径、告警规则和通知渠道。

##### 创建 FIM 策略
1. 导航至 `/security/fim/policies`
2. 点击"新建策略"按钮
3. 输入策略名称和描述
4. 添加监控目标服务器
5. 配置监控路径（文件或目录）
6. 设置告警规则和通知渠道
7. 点击"保存"

##### 执行基线扫描
1. 在策略列表中找到目标策略
2. 点击"基线扫描"按钮
3. 选择目标服务器
4. 点击"开始扫描"
5. 扫描完成后可查看基线快照

#### FIM 执行记录 (/security/fim/executions)
查看 FIM 扫描执行历史记录。

##### 查看执行记录
1. 导航至 `/security/fim/executions`
2. 查看扫描执行列表
3. 根据策略名称、执行状态和时间筛选记录
4. 点击记录可查看执行详情

#### FIM 事件 (/security/fim/events)
查看检测到的文件变更事件。

##### 查看文件变更事件
1. 导航至 `/security/fim/events`
2. 查看文件变更事件列表
3. 根据服务器、策略名称和严重程度筛选
4. 点击事件可查看变更详情

#### FIM 告警 (/security/fim/alerts)
查看 FIM 触发的告警记录。

##### 查看告警
1. 导航至 `/security/fim/alerts`
2. 查看告警列表
3. 根据告警级别、状态和时间筛选
4. 可对告警进行处理或标记

#### FIM 已知主机 (/security/fim/known-hosts)
管理 SSH 主机密钥，用于无代理模式的 FIM 扫描。

##### 添加主机密钥
1. 导航至 `/security/fim/known-hosts`
2. 点击"新增主机"按钮
3. 输入主机名称和 IP 地址
4. 选择主机类型（物理机、虚拟机）
5. 粘贴 SSH 公钥内容
6. 点击"保存"

## 8. 系统管理

### 8.1 用户管理 (/admin/users)
用户管理模块负责管理平台的所有用户账号。

#### 新增用户
1. 导航至 `/admin/users`
2. 点击"新建用户"按钮
3. 输入用户基本信息（用户名、姓名、邮箱）
4. 分配角色和权限
5. 设置初始密码
6. 点击"保存"

#### 用户权限
- **系统管理员**：完全访问权限
- **运维工程师**：资产中心、变更发布、监控中心权限
- **开发人员**：查看权限，有限的部署权限
- **安全管理员**：安全中心相关权限

### 8.2 角色管理 (/admin/roles)
角色管理用于定义不同用户角色及其权限。

#### 创建角色
1. 导航至 `/admin/roles`
2. 点击"新建角色"按钮
3. 输入角色名称和描述
4. 分配权限（菜单、操作、数据范围）
5. 点击"保存"

### 8.3 菜单管理 (/admin/menus)
菜单管理允许管理员自定义用户界面菜单结构。

#### 自定义菜单
1. 导航至 `/admin/menus`
2. 点击"编辑菜单"按钮
3. 添加、修改或删除菜单项
4. 设置菜单可见性和访问权限
5. 保存菜单配置

### 8.4 平台审计 (/platform/audit)
平台审计模块记录和审计用户在平台上的所有操作行为。

#### 审计日志类型
- **操作日志**：记录用户在平台上的所有操作（创建、修改、删除等）
- **访问日志**：记录用户访问平台 API 的情况
- **登录日志**：记录用户的登录和登出行为

#### 查看审计日志
1. 导航至 `/platform/audit`
2. 选择审计类型标签页（概览、操作、访问、登录）
3. 使用筛选条件搜索目标日志
4. 可按用户名、模块、操作类型、状态、IP 地址和时间范围筛选
5. 点击日志可查看详细信息

#### 日志归档与清理
平台支持审计日志的归档和清理功能，可配置日志保留策略。

##### 配置日志保留策略
1. 在审计页面找到"日志保留"设置
2. 设置日志保留天数
3. 选择清理方式：仅归档、仅清理在线数据、归档并清理
4. 点击"执行"完成设置

### 8.5 我的资料 (/profile)
我的资料页面用于查看当前登录用户的个人信息和安全设置。

#### 修改密码
1. 点击右上角用户下拉菜单中的“我的资料”，或直接访问 `/profile`
2. 在“安全设置”区域找到“登录密码”
3. 点击“修改密码”按钮
4. 输入当前密码
5. 输入新密码并再次确认新密码
6. 点击确定完成修改

## 10. 最佳实践

### 11.1 项目组织
- 按业务线或产品线创建项目
- 为不同环境设置明确的命名规范
- 定期清理不用的资源

### 11.2 安全规范
- 定期更新系统和应用补丁
- 实施最小权限原则
- 定期进行安全审计

### 11.3 监控建议
- 为关键业务指标设置告警
- 定期审查告警规则，避免告警疲劳
- 建立清晰的事件响应流程

### 11.4 Jenkins管理建议
- 合理使用视图复制功能，避免重复创建相同Job
- 定期清理不再使用的Jenkins视图和Job
- 妥善保管SSH凭据，定期更新密钥
- 利用聚合打包功能提高部署效率
