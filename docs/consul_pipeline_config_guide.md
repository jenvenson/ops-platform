# 批量配置下发功能使用指南

## 概述
批量配置下发功能允许您批量管理和复制 Consul 中的配置，特别适用于多环境部署场景，如从 demo 环境复制配置到生产环境。

## 功能特点
- 批量复制 Consul 配置
- 支持多环境配置管理
- 灵活的替换规则
- 一键复制所有项目

## 使用步骤

### 1. 选择 Consul 配置
- 从下拉菜单中选择要操作的 Consul 实例
- 确保所选配置可正常访问

### 2. 设置源后缀和目标后缀
- 源后缀：指定要从哪个环境复制（如：`demo`）
- 目标后缀：指定要复制到哪个环境（如：`prod`、`147`）
- 示例：`plugin/*/demo` → `plugin/*/prod`

### 3. 选择复制选项
- 递归复制：选择是否复制所有子键
- 项目选择：可选择特定项目或复制所有项目

### 4. 配置替换规则（可选）
支持多种类型的精确替换：

#### Tag 替换
- 用途：替换版本标签
- 格式匹配：`tag: "V2.5.0"`、`tag: V2.5.0`、`tag='V2.5.0'`、`tag=V2.5.0` 等
- 示例：`V2.5.0` → `V2.5.1`，`V2.5.0` → `test10`

#### Server 替换
- 用途：替换服务器地址
- 格式匹配：`server: "10.70.70.231"`、`server: 10.70.70.231`、`server="10.70.70.231"` 等
- 示例：`10.70.70.231` → `10.70.70.111`

#### Branch 替换
- 用途：替换分支名称
- 格式匹配：`branch: "develop"`、`branch: main` 等
- 示例：`develop` → `main`

#### SubmoduleBranch 替换
- 用途：替换子模块分支
- 格式匹配：`submoduleBranch: "feature-branch"` 等
- 示例：`feature-branch` → `release-v1.0`

### 5. 高级替换规则
- 文本替换：简单字符串替换
- 正则替换：使用正则表达式进行复杂替换

### 6. 执行配置
- 点击"执行批量配置下发"按钮开始执行
- 系统将显示复制进度和结果

## 使用示例

### 示例1：版本升级
- 源后缀：`demo`
- 目标后缀：`prod`
- Tag 替换：`V2.5.0` → `V2.5.1`

### 示例2：环境迁移
- 源后缀：`test`
- 目标后缀：`staging`
- Server 替换：`10.70.70.231` → `10.70.70.111`

### 示例3：多环境复制
- 从：`plugin/app/demo`
- 到：`plugin/app/prod`
- 使用多种替换规则同时更新版本、服务器地址等

## 注意事项

1. **替换规则精度**：
   - 系统会精确匹配键值对格式，不会误替换其他内容
   - 只替换符合指定格式的键值对，如 `tag: "value"`，不会替换纯文本中的 `value`

2. **备份策略**：
   - 建议在执行重要操作前备份相关配置
   - 系统会记录操作历史，便于回滚

3. **权限要求**：
   - 需要有对应 Consul 实例的读写权限
   - 某些敏感配置可能需要额外授权

4. **配置验证**：
   - 执行替换后请验证目标环境配置的正确性
   - 检查是否有遗漏或错误的替换

## 故障排除

### 常见问题
1. **替换不生效**：检查值的格式是否与预期匹配（带引号 vs 不带引号）
2. **权限不足**：联系管理员获取相应 Consul 实例访问权限
3. **网络问题**：确保与 Consul 服务器的网络连通性

### 验证方法
- 手动检查 Consul 中的目标键值
- 查看操作日志确认替换结果
- 验证应用是否正确读取新配置

## 最佳实践

1. **分阶段执行**：先在测试环境验证配置
2. **逐步替换**：对于复杂替换，考虑分多次执行
3. **详细记录**：记录每次替换的目的和影响范围
4. **实时验证**：替换后立即验证应用行为

通过遵循本指南，您可以高效、安全地使用批量配置下发功能，实现跨环境的配置管理和部署。
