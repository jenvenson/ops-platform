# 运维小助手开发待办清单

## 1. 说明

本文档用于整理 OPS Platform “运维小助手”后续开发任务，按优先级拆分为：

- `P0`：一期上线前必须完成
- `P1`：一期可用后应尽快补齐
- `P2`：中期增强能力

当前已完成的基础能力：

- 前端聊天入口已切到 `/api/assistant/*`
- 后端已建立 `assistant` 模块骨架
- 已接入 `Ollama + qwen2.5:1.5b`
- 已支持文档问答、页面导航
- 已支持部分只读工具：
  - CMDB 项目 / 环境 / 主机
  - 部署历史 / 聚合历史
  - 告警事件
- 已支持结构化返回：
  - `citations`
  - `actions`
  - `resultCards`

## 2. P0：上线前必须完成

### 2.1 鉴权与身份绑定

- assistant 会话必须绑定当前登录用户
- 禁止匿名调用 assistant 查询接口
- 工具调用必须继承当前用户权限上下文
- 前端未登录状态不显示或不允许使用运维小助手

### 2.2 会话与历史能力

- 增加会话历史查询接口
- 支持查看当前会话历史消息
- 支持重新打开历史会话
- 增加会话标题或摘要字段

### 2.3 错误处理与降级

- Ollama 不可用时统一返回标准错误结构
- 工具查询失败时返回可解释错误信息
- 数据库超时、空结果、内部错误需区分处理
- 避免前端把所有错误都显示成“AI 暂时无法响应”

### 2.4 RAG 真检索第一版

- 将当前关键词检索升级为文档分片检索
- 建立文档索引结构
- 支持按标题、模块、路径召回
- 优化片段摘要质量，减少无关引用

### 2.5 可观测性

- 记录模型调用耗时
- 记录工具调用耗时
- 记录错误次数
- 记录命中工具类型
- 记录 assistant 请求量

### 2.6 基础安全保障

- 增加接口限流
- 限制单次问题长度
- 限制会话上下文长度
- 禁止模型返回敏感配置或密钥信息

## 3. P1：一期完成后优先补齐

### 3.1 安全模块查询

- 查询扫描任务
- 查询资产中心
- 查询漏洞列表
- 查询漏洞详情摘要

### 3.2 Jenkins / 发布联动查询

- 查询 Jenkins Job 最近构建状态
- 查询某次部署失败原因
- 查询某应用最近发布记录
- 查询某次聚合打包详情

### 3.3 告警查询增强

- 支持按级别筛选
- 支持按状态筛选
- 支持按主机 / 来源筛选
- 支持按规则名模糊查询

### 3.4 前端体验增强

- 增加“建议提问”快捷入口
- 增加结果卡片点击跳转
- 增加引用折叠 / 展开
- 增加调试标签显示当前模型和数据来源
- 增加更清晰的错误态和空态

### 3.5 Prompt 优化

- 为知识问答、导航、只读查询拆分独立 Prompt
- 调整回答长度控制
- 降低模型幻觉
- 对回答格式做更严格约束

## 4. P2：中期增强

### 4.1 向量检索正式版

- 接入本地 embedding 模型
- 建立向量索引
- 文档增量更新机制
- 支持按模块和标签过滤检索

### 4.2 上下文管理增强

- 支持多轮会话摘要
- 长会话压缩
- 用户偏好记忆
- 常见问题缓存

### 4.3 工具编排增强

- 支持多工具串联查询
- 支持先查再总结的多步流程
- 返回更明确的 `toolName / sourceType / confidence`
- 工具结果结构标准化

### 4.4 结果展示增强

- 表格式结果卡片
- 告警/部署/CMDB 专属卡片模板
- 搜索结果分组展示
- 支持复制结果、导出摘要

## 5. 暂不建议过早开发

以下能力不建议在当前阶段过早投入：

- 自动发布
- 自动回滚
- 自动修改 Consul 配置
- 自动执行 Shell / SQL
- 完全自治 Agent

原因：

- 当前权限、审计、确认机制还未完善
- 运维执行类动作风险高
- 一期更适合先做“能答、能查、能带路”

## 6. 推荐开发顺序

建议按以下顺序推进：

1. 完成鉴权、历史、降级、限流
2. 完成 RAG 真检索第一版
3. 接入安全模块查询
4. 接入 Jenkins / 发布联动查询
5. 优化 Prompt 与前端交互
6. 再评估执行类能力

## 7. 验收标准建议

满足以下条件可认为一期达到“可用”状态：

- 用户可稳定发起会话并获得响应
- 文档问答能返回真实引用
- 页面导航能返回正确入口
- CMDB / 部署 / 告警查询能返回真实结果
- 接口失败时有明确错误提示
- 会话与工具调用可追踪
- 不暴露敏感信息

## 8. 当前最建议的下一步

如果只选一个最值得继续推进的方向，建议优先做：

- `鉴权 + 会话历史 + RAG 真检索`

这三项补齐后，运维小助手才会从“可演示原型”更接近“可上线使用的一期系统”。
