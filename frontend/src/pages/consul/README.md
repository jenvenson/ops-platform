## Consul 配置使用说明

Consul 配置功能允许用户添加、编辑、测试和管理多个 Consul 服务器的连接配置。通过此功能，您可以轻松管理不同环境（如开发、测试、生产）的 Consul 服务。

### 功能特点

- **多配置支持**：可以添加多个不同的 Consul 服务实例
- **连接测试**：可以测试配置的连接性
- **默认配置**：可以设置一个默认的 Consul 配置
- **安全认证**：支持 ACL Token 和基本认证
- **配置操作记录**：查看所有复制操作的历史记录
- **高级替换规则**：支持自定义替换规则，包括branch、server等特定key的替换

### 字段说明

1. **配置名称**：为这个 Consul 配置起一个易于识别的名字（如："生产环境Consul"）
2. **Consul地址**：Consul 服务的访问地址（如：`http://127.0.0.1:8500`）
3. **数据中心**：Consul 数据中心名称（默认为 `dc1`）
4. **ACL Token**：（可选）用于认证的 ACL Token
5. **用户名/密码**：（可选）基本认证凭据

### 使用步骤

1. **添加配置**
   - 点击页面右上角的"添加配置"按钮
   - 输入配置信息并保存

2. **编辑配置**
   - 在配置列表中找到目标配置
   - 点击"编辑"按钮进行修改

3. **测试连接**
   - 在配置列表中点击"测试连接"按钮
   - 验证配置是否正确

4. **设置默认配置**
   - 点击"设为默认"按钮将此配置作为默认使用

5. **删除配置**
   - 点击"删除"按钮可移除不再需要的配置

### 批量复制功能

#### 基本复制
1. 选择源后缀和目标后缀（如从 "demo" 复制到 "147"）
2. 选择是否递归复制
3. 点击"执行批量配置下发"

#### 高级替换规则
1. 使用"简单替换规则"进行基本的字符串替换
2. 使用"高级替换规则"添加复杂的文本替换或正则表达式替换
3. 特别支持branch、server等key的替换

#### 查看操作历史
1. 使用"配置操作记录"菜单项查看所有历史操作
2. 在配置操作记录页面查看详细的操作结果和状态

### 安全注意事项

- 请妥善保管 ACL Token 和认证凭据
- 建议为不同的环境使用不同的配置
- 定期检查和更新过期的认证信息
